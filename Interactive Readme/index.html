Project Overview

This is a complete, secure, and scalable Expense Reimbursement Web Application built using a modern microservices architecture. It's powered by React for the frontend and Spring Boot with Spring Cloud for the backend, utilizing MySQL for persistent data storage. This system facilitates a robust workflow where employees can submit expense requests, and managers can securely review, approve, or reject them.

Key Features

The application provides a distinct set of features based on user roles, all secured by a central authentication system. Explore the different capabilities below.

Authentication & Authorization

  • **Secure Login / Registration:** Dedicated `AuthService` handles user authentication.
  • **Manager Registration:** Managers must register using a secret code: `RAJ`.
  • **Role-Based Screen Rendering:** UI components dynamically render based on the user's role (Employee or Manager).
  • **Protected Routes:** Both frontend (React Router) and backend (API Gateway) secure routes based on user session and role.
  • **Session Management:** User session details (`userId`, `role`) are securely stored in `localStorage`.
  • **Axios Interceptors:** Automatically injects necessary user headers (`userId`, `role`) into every API request for backend validation.

Employee Features (Role: `EMPLOYEE`)

  • **Expense Submission:** Employees can submit new expense reimbursement requests.
  • **View Own Data:** Can only view expenses they have personally submitted.
  • **Modification Policy:** Can only **edit** and **delete** expenses that are currently in the `PENDING` status.
  • **Data Isolation:** Cannot access, modify, or submit data for any other employee.

Manager Features (Role: `MANAGER`)

  • **Global View:** Can view **all** submitted expenses across the organization.
  • **Actionable Tasks:** Can **Approve** or **Reject** any pending expense request.
  • **Employee Directory:** Access to view and search the employee directory.
  • **Data Immutability:** Managers **cannot** add, edit, or delete expense requests, only change their status.

Security Highlights

  • **Backend Validation:** Every API call is validated by the relevant microservice, checking both the user's role and their identity (`userId`).
  • **Cross-Service Authorization:** An employee attempting to access `/expenses/{otherId}` will be blocked.
  • **Data Integrity:** Employees cannot submit an expense request on behalf of another user.
  • **CORS Management:** The API Gateway handles all Cross-Origin Resource Sharing (CORS) configurations centrally.
  • **Service Discovery Security:** All microservices are registered in the private Eureka registry, ensuring they are only reachable via the secured API Gateway.

How to Run the Project

Follow these steps sequentially to get the entire application running locally.

1 Setup Databases

Ensure you create three separate MySQL schemas: `er_auth_db`, `er_employee_db`, and `er_expense_db`. Update the `application.properties` in each service with your local MySQL credentials.

2 Start Eureka Server

This must be started first. Navigate to `Backend/EurekaServerER` and run `mvn spring-boot:run`.

  • `Backend/EmployeeServiceER`
  • `Backend/ExpenseServiceER`

4 Start API Gateway

Navigate to `Backend/APIGatewayER` and run `mvn spring-boot:run`.

Future Enhancements

This project provides a solid foundation. Future work could include these enhancements:

  • **Advanced Authentication:** Implement **JWT** (JSON Web Token) for stateless authentication.
  • **Deployment:** Containerize the entire application using **Docker** and **Docker Compose**.
